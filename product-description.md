# AI 短剧知识库系统产品描述

## 一、产品定位

本系统不是单纯的后台数据库，而是一套面向创作者的可交互创作系统。它通过结构化知识、趋势洞察和 AI 共创能力，支撑编剧在“人与 AI 协作”的环境下完成选题、人物设定、剧情设计、剧本创作，并与下游生图、生视频系统衔接。

当前核心内容形态是 AI 短剧：单集 1-2 分钟，总时长约 90-120 分钟。系统策略是“短剧优先，底层泛化”：优先把 AI 短剧做深，同时保留未来扩展到知识商业类、口播类、广告类、图文转视频等内容形态的能力。

## 二、核心原则

1. 知识库不是资料堆，而是可调用的创作参考系统。
2. 前端体验不应像后台管理系统，而应像创作者驾驶舱。
3. 底层数据结构保持通用，应用层围绕 AI 短剧深度专业化。
4. 系统重点解决 AI 创作中的空泛、风格漂移、人物崩坏、剧情重复、长篇连续性不足等问题。
5. 每个模块都应服务于创作闭环：趋势发现 -> 选题判断 -> 剧情生成 -> 分镜提示词 -> 生图/生视频交付。

## 三、核心功能模块

| 模块 | 作用 | 典型能力 |
|---|---|---|
| 知识采集与入库 | 把外部资料变成可用素材 | 手动录入、批量导入、来源管理、去重、基础标签 |
| 知识结构化 | 把资料变成创作积木 | 题材、情绪、冲突、人物关系、桥段、台词、场景、风格标签 |
| 短剧素材资产库 | 沉淀可复用创作资产 | 选题库、人物库、情节库、桥段库、台词库、情绪库、案例库 |
| 检索与推荐 | 让编剧快速找到参考 | 关键词检索、语义检索、标签筛选、相似案例、创作推荐 |
| 选题工作台 | 从灵感到可开发选题 | 选题生成、题材筛选、受众匹配、爆点分析、风险检查 |
| 剧本共创工作台 | 人和 AI 协作生成剧本 | 大纲、分集、人物、冲突、台词、节奏、版本对比、局部重写 |
| 连续性管理 | 支撑 90-120 分钟长篇短剧 | 前情、伏笔、人物状态、关系变化、未解决冲突、已用桥段 |
| 下游生成衔接 | 连接生图和生视频系统 | 分镜拆解、角色设定、场景设定、画面提示词、视频提示词 |
| 质量评估与迭代 | 让系统越用越准 | 剧本评分、可拍性检查、风格一致性检查、反馈回流 |

## 四、选题雷达模块

选题雷达是系统的外部趋势入口，目标是给编剧提供选题参考和方向引导。它不只是榜单页，而是“市场情绪感知器”，负责把外部热度、评论情绪和社会话题转化为可开发的短剧选题。

核心流程：热门内容采集 -> 评论情绪解析 -> 观众需求洞察 -> 时代情绪母题提炼 -> 短剧选题建议。

| 子模块 | 关注问题 | 输出结果 |
|---|---|---|
| 热门短剧榜 | 各主要视频平台最近哪些短剧火 | 平台、排名、题材、热度、增长趋势、核心卖点 |
| 评论情绪洞察 | 观众为什么喜欢、吐槽或共鸣 | 高频评论、情绪分类、爽点、雷点、遗憾点、潜在需求 |
| 跨平台话题扫描 | 短剧外还有哪些热点可转化 | 图文、文字、长视频、社媒话题、可短剧化角度 |
| 时代情绪母题 | 当前人群底层情绪是什么 | 阶层焦虑、关系失控、普通人反击、女性夺权、身份逆袭等母题 |
| 选题生成器 | 如何把洞察变成项目 | 题材建议、人设组合、核心冲突、前三集钩子、风险点、推荐指数 |

选题雷达的最终产物应是“选题卡”，而不是单纯报告。选题卡建议包含：来源趋势、代表作品、观众情绪、底层需求、时代母题、可短剧化方向、推荐题材、人设组合、核心冲突、前三集钩子、风险点、相似题材饱和度、推荐指数。

## 五、MVP 范围

第一阶段不追求全品类视频创作平台，而是先打通 AI 短剧创作闭环：短剧项目库 -> 选题雷达 -> 素材结构化 -> 世界观/人物库 -> 分集大纲 -> 单集剧本共创 -> 分镜与提示词导出 -> 质量评估。

MVP 的验证目标是：知识库是否能明显提升 AI 短剧剧本的具体性、连续性、情绪强度、可拍性和风格稳定性。

## 六、产品方向总结

本系统应定义为“短剧优先的多内容形态创作知识库”。短期用 AI 短剧打磨专业深度，长期用抽象知识底座扩展到更多视频内容类型。它的核心竞争力不在于存了多少资料，而在于能否把外部趋势、观众情绪、结构化素材和 AI 生成能力连接成一条稳定的创作生产线。
