# 创意库 Qwen 真实解析 MVP 设计

## 目标

在不引入数据库的前提下，把现有“创意库”原型升级为可真实上传视频、调用阿里云百炼 Qwen、生成结构化元数据并进行人工审核的本地应用。

## 架构

`浏览器页面 -> FastAPI 本地服务 -> Qwen3.5-Omni -> Pydantic 校验 -> 本地 JSON`

- 页面保留现有创意库和选题助手的视觉与交互。
- FastAPI 负责保护 API Key、接收文件、执行长任务和提供结果接口。
- Qwen3.5-Omni 通过百炼 OpenAI 兼容接口读取视频画面与声音。
- 不使用数据库；任务、原始解析结果和人工修订结果写入 `runtime-data/`。

## 本轮范围

1. 模型配置：Base URL、API Key、模型 ID、连接测试；密钥只保存在服务内存，不写入浏览器或配置文件。
2. 视频上传：MP4、MOV、MKV、AVI、WEBM，单文件最大 80MB；字幕与评论文件可选。
3. 单视频解析：覆盖现有页面 54 项视频元数据、分镜时序表和观众情绪曲线。
4. 证据约束：每项包含结论、证据、时间范围、置信度、适用条件、失效边界和审核状态。
5. 数据边界：没有评论或行为数据时，观众反馈字段必须标为 `unknown`，不得生成量化指标。
6. 本地持久化：分析结果保存为 JSON，支持页面人工修改后同步回写。

## 非目标

- 不实现数据库、账号权限、多人协作和云端部署。
- 不实现整剧多集聚合解析；整剧页面继续保留现有展示结构。
- 不抓取平台评论、播放量、留存率等外部数据。
- 不把 API Key 保存在 `localStorage`、JSON 或 Git 中。

## 失败处理

- 未配置或连接失败时禁止开始解析，并显示明确错误。
- 上传格式或大小不合法时立即拒绝。
- 模型输出无法通过结构校验时自动执行一次文本修复；仍失败则保留任务错误和原始响应摘要。
- 服务重启后可从任务 JSON 恢复已完成和失败任务；运行中的任务标记为可重试。

## 验收

- 用户可在页面完成配置、连接测试、视频上传、进度查看和结果审核。
- 解析结果包含全部 54 项视频字段，且观众数据缺失时不编造。
- 分镜表和情绪曲线使用模型返回的数据，不再使用演示模板。
- API Key 不落盘；任务和结果 JSON 可检查、导出和恢复。
